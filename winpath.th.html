-โปรแกรมบนลินุกส์เป็นแบบ share library (โปรแกรมยืมโปรแกรม), ส่วนบนวินโดวส์เป็นแบบ stand alone (โปรแกรมเดียวมีครบ)

หมายความว่า วินโดวส์นั้น ไฟล์โปรแกรม (ไม่นับ config)  จะอยู่รวมกันในที่เดียว คือ C:\Program Files, แต่ลินุกส์มีไฟล์หลักของ 1 โปรแกรมกระจายกันอยู่ในหลายที่ของ /usr (ใช้ทุกยูเซอร์) และ /home/<username> (ใช้ได้เฉพาะยูเซอร์นั้นๆ) .
เช่น winPath บน Windows จะมีไฟล์ทั้งหมดอยู่ใน C:\Program Files\winPath
แต่ winPath บน Linux จะมีไฟล์กระจายอยู่ในที่เหล่านี้ /computer, /usr/bin, /usr/lib, /usr/share/winPath, /usr/share/icons

-เครื่องหมาย "~" หมายถึง /home/<username> เช่น ชื่อล็อกอินเป็น winpath ~ ก็จะเท่ากับ /home/winpath

-ทำความเข้าใจกับบ้านของนาย root

นาย root เป็นเจ้าของหมู่บ้านและตึกชื่อ root ( / ) ที่มีบ้านเก็บข้าวของแบ่งเป็นหมวดไว้หลายหลัง คือ /bin, /debian, /etc, /lib, /lost+found, /opt, /run, /srv, /usr, /boot, /dev, /home, /lib32, /media, /proc, /sbin, /sys, /var, /computer, /lib64, /mnt, /root, /selinux, /tmp.
ต่อมานาย root ได้เปิดคอนโดของเขาแห่งหนึ่งชื่อ /home เป็นอพาร์ทเม้นต์ ให้ผู้ใช้ (user) รายอื่นๆ มาทำเป็นบ้านส่วนตัว โดยอนุญาตให้ตังชื่อห้องเป็นชื่อตัวเอง (username) ได้. ซึ่งนาย root เอง เรียกทุกหลังรวมๆ กันว่า home เช่น home ของ a คือห้อง /home/a เป็นต้น. นอกจากนี้ยังมีกฎในการใช้ข้าวของในบ้านต่างๆ ร่วมกับผู้ใช้อื่นๆ ด้วย.
โดยมากแล้วผู้คนในหมูบ้านนาย root มักจะเลียนแบบวิธีเก็บข้าวของที่ต้องใช้ร่วมกับของๆ นาย root ไปด้วย แต่มักจะเก็บไว้ในห้องของตัวเอง แต่ใส่ไว้ในลิ้นชักที่มีจุดนำหน้า (.) เพื่อไม่ให้รกห้องของตัวเอง เช่น ~/.<ชื่อโปแกรมที่ใช้ร่วม>, บางทีทำลิ้นชักเลียนแบบคลังโปรแกรม (/usr/share) ของนาย root ไปเลยเป็น ~/.local/share/<ชื่อโปรแกรม>.

ผู้ใช้ใช้ของร่วมกับนาย root ได้

ผู้ใช้ทุกรายสามารถเปิดใช้ข้าวของในบ้านหลังอื่นๆ ร่วมกับของนาย root ได้ เท่าที่นาย root ประกาศิต (permission) ไว้ โดยนาย root จะเขียนป้านติดคำอนุญาตไว้ เป็นเลข 3 หลัก สำหรับคน 3 กลุ่ม คือ หลักแรกหมายถึง ตัวเอง (owner), หลักที่สองหมายถึง ก๊ก (group), หลักที่สามหมายถึง สัตว์ทั่วไปในไตรภพ (others).
เลข แต่ละหลักบวกรวมกันจะได้ไม่เกิน 7 คือ
1 - ใช้ทำงานการได้ แต่ห้ามอ่าน คัดลอก เลียนแบบ แก้ไข ลบทิ้ง ไม่ได้ทั้งนั้น
2 - แก้ไขได้ ลบทิ้งได้ แต่อ่าน คัดลอก เลียนแบบ หรือแม้แต่เอาไปใช้ทำงานก็ไม่ได้
4 - อ่านได้ คัดลอกได้ เลียนแบบได้ แต่ยังไม่อนุญาติให้เอาต้นฉบับไปแก้ไข ลบทิ้ง หรือ แม้แต่ใช้ทำงาน

เลขเหล่านี้ นาย root จะเอามาบวกกัน ถ้าเขาอยากอนุญาตให้ทำได้มากกว่า 1 อย่าง

ตัวอย่างเลข เช่น:
นาย root แปะโน้ต ไว้บนสูตรทำอาหาร 755
7 ตัวแรกมาจาก 1+2+4    ให้ตัวนาย root เองใช้งานได้ + แก้ไขได้ + อ่านสูตรได้
5 ตัวที่สองมาจาก 1+4     ให้ก๊กที่นาย root กำหนดไว้ ใช้งานได้ + อ่านสูตรได้ (เท่าที่เปิดให้อ่าน)
5 ตัวที่ 3 ก็เหมือนกัน แต่อันนี้อนุญาตให้สรรพสัตว์ทั้งปวง
เป็นต้น

บ้านหลังต่างๆ ของนาย root: ความหมายของ Directory ต่างๆ บน Linux

/bin : เป็นไดเร็คทอรีที่ใช้เก็บคำสั่งทัวไป รวมทั้งคำสั่ง utilities ของระบบ
/boot : เป็นไดเร็คทอรี่ที่เก็บ kernel และไฟล์ที่ใช้ Boot ของระบบ
/dev : เป็นที่เก็บ device file สำหรับใช้อ้างอิงถึงตัวอุปกรณ์ โดยอุปกรณ์ทุกตัวจะมีไฟล์เป็นของตัวเองเช่น /dev/tty0 ก็คิอ serial port com1
/etc : เป็นไดเร็คทอรี่ที่สำคัญมากใช้เก็บคำสั่งเพิ่มเติม และใช้เก็บไฟล์ข้อมูลประเภท Config และ ไฟล์หลักๆ ที่ใช้ setup ระบบ เช่น rc.local
/home : เป็นที่ใช้เก็บข้อมูลของ User ในระบบ
/lib : เป็นที่ใช้เก็บ Libary แบบ Dynamic ที่เขียนจากภาษา C มีลักษณะคล้าย .dll ของ Windows
/lost+found : เป็นที่ใช้เก็บ error ทั่วไปเกี่ยวกับ Disk
/media : เป็นที่เก็บ device file สำหรับอ้างอิงถึงอุปกรณ์ทุกตัวที่จะมีไฟล์เป็นของตัวเอง cdrom (/media/cdrom) หรือ harddisk (/media/win) เป็นต้น
/proc : เป็นที่สำหรับเก็บข้อมูลขณะ Process ต่างๆกำลังทำงานอยู่
/sbin : เป็นที่ใช้เก็บคำสั่งในการดูแลระบบต่างๆ
/tmp : เป็นที่ใช้เก็บไฟล์ชั่วคราว
/usr : เป็นไดเร็คทอรีย่อยในการ setup ระบบ
/usr/bin : เก็บคำสั่งทั่วไป
/var : เป็นที่ใช้เก็บข้อมูลที่เปลี่ยนแปลงตลอดเวลา อย่างเช่น /www  ซึ่งเป็นที่เก็บ html php ไฟล์   ,/log เป็นที่เก็บ log ต่างๆ
